# CLI Commands Contract

**Feature**: 001-automated-test-framework  
**Created**: 2026-04-08  
**Purpose**: 定义CLI命令接口规范

## 命令概览

| 命令 | 用途 | 参数 | 输出格式 |
|------|------|------|----------|
| `coredump list devices` | 列出所有待测设备 | 无 | 表格 |
| `coredump list testcases` | 列出所有测试用例 | 无 | 表格 |
| `coredump list tasks` | 列出所有测试任务 | 无 | 表格 |
| `coredump run <task_id>` | 执行指定的测试任务 | task_id | 进度 + 结果 |

## 命令详细规范

### 1. coredump list devices

**功能**: 列出所有待测设备的基本信息

**命令格式**:
```bash
coredump list devices
```

**参数**: 无

**输出格式**: 表格

```
ID          Name            IP Address       Port    Username
----------  --------------  ---------------  ------  ----------
device001   测试设备1        192.168.1.100    22      root
device002   测试设备2        192.168.1.101    22      root
```

**字段说明**:
- **ID**: 设备唯一标识符
- **Name**: 设备名称
- **IP Address**: IP地址
- **Port**: SSH端口
- **Username**: SSH用户名

**错误处理**:
- 配置文件目录不存在或为空：显示"无待测设备配置"
- 配置文件格式错误：显示错误详情

**退出码**:
- 0: 成功
- 1: 失败

---

### 2. coredump list testcases

**功能**: 列出所有测试用例的基本信息

**命令格式**:
```bash
coredump list testcases
```

**参数**: 无

**输出格式**: 表格

```
ID          Name                    Scenarios    Timeout    On Failure
----------  ----------------------  -----------  ---------  -----------
tc001       网络连通性测试           1            30         stop
tc002       服务状态检查             2            60         continue
```

**字段说明**:
- **ID**: 测试用例唯一标识符
- **Name**: 测试用例名称
- **Scenarios**: 包含的测试场景数量
- **Timeout**: 命令执行超时时间（秒）
- **On Failure**: 失败后行为（continue或stop）

**错误处理**:
- 配置文件目录不存在或为空：显示"无测试用例配置"
- 配置文件格式错误：显示错误详情

**退出码**:
- 0: 成功
- 1: 失败

---

### 3. coredump list tasks

**功能**: 列出所有测试任务的基本信息

**命令格式**:
```bash
coredump list tasks
```

**参数**: 无

**输出格式**: 表格

```
ID          Name            Devices    Testcases    Report Format    Status
----------  --------------  ---------  -----------  ---------------  --------
task001     网络测试任务     2          2            html             pending
task002     系统测试任务     1          3            json             running
```

**字段说明**:
- **ID**: 测试任务唯一标识符
- **Name**: 测试任务名称
- **Devices**: 目标设备数量
- **Testcases**: 包含的测试用例数量
- **Report Format**: 报告格式（json或html）
- **Status**: 任务状态（pending、running、completed、failed）

**错误处理**:
- 配置文件目录不存在或为空：显示"无测试任务配置"
- 配置文件格式错误：显示错误详情

**退出码**:
- 0: 成功
- 1: 失败

---

**命令格式**:

coredump run <task_id>

## 全局选项

### 帮助信息

```bash
coredump --help
```

显示所有可用命令和选项。

### 版本信息

```bash
coredump --version
```

显示框架版本信息。

### 配置文件路径

```bash
coredump --config-dir /path/to/config list devices
```

指定配置文件目录路径（默认: `config/`）。

### 报告输出路径

```bash
coredump --report-dir /path/to/reports run task001
```

指定报告输出目录路径（默认: `reports/`）。

## 实现要求

### 日志级别规范

**使用规则**:
- **INFO**: 正常执行流程信息（开始、结束、阶段切换、命令执行结果）
- **WARNING**: 警告信息（SSH连接失败跳过设备、检查规则失败、继续执行）
- **ERROR**: 错误信息（配置验证失败、任务不存在、并发冲突）

**日志格式**:
```
[YYYY-MM-DD HH:MM:SS] [LEVEL]   消息内容
```

**对齐要求**:
- 时间戳固定宽度: `[YYYY-MM-DD HH:MM:SS]`
- 日志级别固定宽度: `[INFO]   `、`[WARNING]`、`[ERROR]  `
- 消息内容左对齐，不使用层次缩进

### 用户体验
- 使用彩色输出区分不同类型的信息（INFO=绿色、WARNING=黄色、ERROR=红色）
- 命令执行结果在一行显示：`执行命令[通过/失败]: <command>`
- 检查规则结果在一行显示：`检查规则[通过/失败]: <rule>`
- 错误信息包含具体位置和修复建议
- 支持Ctrl+C中断任务执行

### 性能要求
- 列表命令响应时间 < 1秒
- 任务执行实时显示进度
- 大量设备/用例时支持分页显示

### 日志记录
- 所有CLI操作记录到日志文件
- 包含时间戳、命令、参数、结果
- 日志文件存储在 `logs/coredump.log`

### 国际化
- 所有输出使用中文
- 支持UTF-8编码
- 时间戳格式: YYYY-MM-DD HH:MM:SS
